 Все стадии pod install (path_provider_foundation, shared_preferences_foundation, sqflite_darwin, url_launcher_ios и т.д.) прошли успешно, pod'ы установились. Билд падает на этапе Xcode build с exit code 65, не оставляя стек-трейса или сообщения о причинах!

Почему так происходит?
Проблема случается на этапе встроенных build phase Pod'ов (Pods targets), а не в твоем коде.

Нет явных строк с “Missing file”, “Undefined symbols”, “CodeSign”, “Permission denied”. Это косвенно указывает на внутренний баг runner, сбойную работу симлинков или race condition во временных build-артефактах Pods/DerivedData.

Подобные проблемы на GitHub Actions случаются из-за остаточных файлов, ошибок workspace/PBXProj, некорректной синхронизации pod'ов или переполнения build-артефактов у бесплатных раннеров.

PROMPT/Рекомендации по исправлению (GitHub Actions)
1. Удаляй не только Pods и DerivedData, но и весь build Flutter/Pods:

text
- name: Hard clean iOS build caches
  run: |
    cd ios
    rm -rf Pods Podfile.lock
    cd ..
    rm -rf build
    rm -rf ios/Flutter/Flutter.framework
    rm -rf ios/.symlinks
    rm -rf ~/Library/Developer/Xcode/DerivedData
    flutter clean
    cd ios
    pod install --repo-update --verbose
2. После чистки — бей билд только флагом --no-codesign:

text
- name: Build iOS app (no codesign)
  run: flutter build ios --no-codesign --verbose
3. Если не поможет — увеличь подробность логов (выше добавить):

text
- name: Enable verbose output
  run: export NSUnbufferedIO=YES
4. Пробуй запускать другой тип runner (например, с другим региональным сервером), так как Github macOS-раннер иногда падает на pod-этапе из-за перегрузки.

Пояснение
90% exit code 65 — это баг работы pod build phase в GitHub Actions/macOS, ошибка корневой структуры build-артефактов, а не твоего кода.

Тебе надо максимально вычищать build/temp, DerivedData, Pods, .symlinks и всегда делать полный чистый pod install.

Если ошибка повторяется даже после чистки — нужно ловить строку (обычно появляется на 10–100 строк выше Process completed with exit code 65) с описанием, например:

file not found

permission denied

undefined symbols

script phase failed

Вывод:
Сделай жесткую чистку перед билдом, добавь подробный лог, при следующем падении зафиксируй, какая именно строка идет перед exit code 65 и пришли ее — по ней можно уже найти точную причину!
